《金渐层桌宠 V2：像真的活猫一样（行为系统）》

目标不是“多几个动画”。

目标是：

让用户感觉这只猫有自己的节奏、注意力、情绪和习惯。

也就是从“会动的素材”升级成“有生命感的桌宠”。

一、V2 的核心思路

V1 解决的是：

能显示
能呼吸
能摆尾巴
能眨眼
能拖拽

V2 要解决的是：

什么时候动
为什么动
动完以后切到什么状态
用户操作后怎么反馈
长时间挂在桌面上会不会显得呆

所以 V2 的本质是：

状态机 + 行为调度器 + 情绪参数 + 环境感知

二、先给你最终效果目标

这只金渐层在桌面上应该表现为：

平时
原地待机呼吸
偶尔眨眼
尾巴轻摆
随机看左看右
偶尔耳朵抖一下
无人打扰一段时间后
走两步
停下观察
坐下
趴下打盹
用户和它互动时
点击有反应
连点会兴奋或炸毛
拖拽会挣扎一下
松手后整理状态
环境变化时
夜晚更容易睡觉
长时间无操作更安静
高频点击更活跃

这就已经很像“真的活猫”。

三、整体架构

建议拆成 4 层：

PetCharacter
├─ Visual Layer        // 分层图片与动画参数
├─ Motion Controller   // 呼吸、摆尾、摇头、走路等动作
├─ State Machine       // idle / walk / sit / sleep / react
└─ Behavior Scheduler  // 定时决定下一步做什么

再加一个：

Mood System            // 开心、平静、警觉、困倦
四、V2 推荐状态设计

不要一开始做太多。先做这 8 个状态就够了。

1. idle 待机

默认状态。

表现：

呼吸
尾巴轻摆
头微动
随机眨眼
2. lookAround 左右观察

像猫在听动静。

表现：

头左右转
耳朵轻抖
眼睛聚焦
3. walk 巡逻

短距离移动。

表现：

左右移动
腿部交替
身体轻微起伏
尾巴摆幅变大
4. sit 坐下

比一直站着更像活物。

表现：

身体切换为坐姿素材
呼吸更平稳
眨眼频率降低
5. sleep 打盹

进入安静状态。

表现：

闭眼
呼吸节奏慢
偶尔尾巴小幅抖动
可加 Zzz
6. happy 开心反馈

用户点击后的正反馈。

表现：

头抬一下
尾巴摆快一点
眼睛短暂变化
可以加轻微弹跳
7. annoyed 烦躁/炸毛

频繁点击后的反馈。

表现：

耳朵后压
尾巴摆动变急
头偏开
持续时间短
8. dragged 被拖拽

用户抓着它移动时。

表现：

身体轻微拉伸
四肢悬空感
松手后恢复
五、状态切换规则

这部分最关键。没有规则，就只是动作堆砌。

你可以先按下面这套做：

idle -> lookAround
idle -> walk
idle -> sit

lookAround -> idle
walk -> idle
sit -> idle
sit -> sleep
sleep -> idle

任意状态 + 点击 -> happy
高频点击 -> annoyed
拖拽开始 -> dragged
拖拽结束 -> idle
六、行为调度器怎么设计

V2 不建议每一帧都“随机”。那样会很乱。

正确做法是：

每隔一段时间，调度一次下一步行为。

例如：

每 6~12 秒决策一次
根据当前状态、情绪、时间段、用户活跃度决定下一步
一个简单可用的权重例子
当前是 idle 时：
50% 继续 idle
20% lookAround
15% walk
10% sit
5% happy 小动作
当前是 sit 时：
50% idle
30% 继续 sit
20% sleep
当前是 sleep 时：
70% 继续 sleep
30% 醒来 idle
七、情绪系统怎么做

不要做太复杂。首版就做 4 个参数。

type Mood = {
  energy: number      // 活跃度
  trust: number       // 对用户互动接受度
  sleepiness: number  // 困意
  alertness: number   // 警觉度
}

范围统一 0~100。

参数变化逻辑
energy 活跃度
随时间缓慢下降
被点击后小幅上升
走路后下降
睡觉后恢复
sleepiness 困意
随时间上升
夜晚提升更快
睡眠后下降
trust 接受度
温和互动上升
高频打扰下降
alertness 警觉度
鼠标靠近、点击、拖拽后升高
静置后回落
情绪对行为的影响

例如：

sleepiness > 70：更容易进入 sleep
energy > 70：更容易 walk / happy
alertness > 60：更容易 lookAround
trust < 30：点击后更容易 annoyed

这样就不再是“纯随机”。

八、时间系统要不要接入

要，而且非常值。

即使只接入本地时间，也会让行为逻辑立刻高级很多。

建议规则
白天
walk 概率稍高
happy 反馈更积极
夜晚
sit / sleep 概率升高
整体动作变慢
深夜
默认更安静
更容易闭眼和趴着

你不需要联网，只用本地时间即可。

九、用户互动系统

V2 的灵魂之一就是互动反馈。

1. 单击

触发：

happy
短促抬头
尾巴摆快一点
2. 连续点击

触发：

annoyed
耳朵压低
头偏开
尾巴快速甩一下

建议定义一个点击计数窗口：

3 秒内点击 1~2 次：开心
3 秒内点击 4 次以上：烦躁
3. 长按

可作为“抚摸”或“按住”的扩展入口。

你后续可以做：

长按进入亲近状态
或弹出菜单
4. 拖拽

拖拽中：

进入 dragged
关闭正常行为调度

拖拽结束：

回 idle
短暂恢复动画
可附带“整理姿态”
十、看鼠标功能怎么做

这个效果非常加分，而且实现不算重。

基本逻辑

获取鼠标相对桌宠的位置：

鼠标在左边：头向左偏
鼠标在右边：头向右偏
鼠标靠近：警觉度上升
注意

不要做成“实时死盯”。

要有限制：

只有鼠标进入桌宠附近区域才跟随
最大转角 ±8° 左右
低通平滑，不要瞬间抽动

这样更像猫。

十一、动作层设计建议

V2 不应该写死一堆互相覆盖的 transform。

建议把动画拆成“通道叠加”。

例如：

bodyOffsetY = breathing + stateOffset + dragOffset
headRotation = idleHead + lookHead + mouseFollowHead + reactHead
tailRotation = idleTail + moodTail + reactTail

也就是说：

同一个部件最终动作 = 多个行为层叠加后的结果

这样系统才可扩展。

4. 嘴巴至少两种

十四、参数建议

给你一套比较稳的初始值。

idle
呼吸幅度：1.5 ~ 3px
头摆动：±2° ~ ±4°
尾巴摆动：±8° ~ ±14°
眨眼间隔：4 ~ 9 秒随机
lookAround
左右看总时长：1.2 ~ 2.5 秒
最大头转角：±6°
walk
一次巡逻持续：2 ~ 5 秒
每次移动距离：40 ~ 160 px
sleep
呼吸周期更慢：2.5 ~ 4 秒/次
眨眼关闭，改为闭眼静置
十五、状态机数据结构建议

你可以让 Claude Code 按这个方向实现：

type PetState =
  | 'idle'
  | 'lookAround'
  | 'walk'
  | 'sit'
  | 'sleep'
  | 'happy'
  | 'annoyed'
  | 'dragged'

interface PetContext {
  state: PetState
  stateStartAt: number
  facing: 'left' | 'right'
  mood: {
    energy: number
    trust: number
    sleepiness: number
    alertness: number
  }
  interaction: {
    clickCountWindow: number
    lastClickAt: number
    isDragging: boolean
  }
}
十六、行为调度伪规则

建议调度器每 1 秒 tick 一次，但不是每秒都切状态。

例如：

每秒更新 mood
每秒检查是否满足当前状态退出条件
每 6~12 秒做一次下一行为决策
用户事件随时可打断状态

这样节奏比较自然。

十七、事件优先级

这个很重要，不然状态会打架。

建议优先级：

dragged > annoyed/happy > sleep > walk/lookAround/sit > idle

意思是：

拖拽中，别做别的
用户强交互优先于自动行为
sleep 一旦进入，不要轻易被普通小动作打断

十九、最终展示效果建议

你后续演示给别人看，不要只说“它会眨眼”。

你要展示这几个场景：

场景 1：静置桌面

它自己呼吸、看左右、偶尔甩尾。

场景 2：鼠标靠近

它会轻微转头看你。

场景 3：点击几次

它先开心，再烦躁。

场景 4：长时间不操作

它慢慢坐下，然后睡觉。

二十、给 Claude Code 的执行清单

你可以直接丢给它：

基于现有 Electron + Vue3 金渐层桌宠项目，升级 V2 行为系统，要求如下：

1. 建立桌宠状态机：
   idle / lookAround / walk / sit / sleep / happy / annoyed / dragged

2. 建立行为调度器：
   - 每秒更新一次内部 context
   - 每 6~12 秒随机决策下一行为
   - 不同状态按权重切换，不要纯随机乱跳

3. 建立 mood 系统：
   energy / trust / sleepiness / alertness
   并让 mood 影响状态切换概率

4. 建立互动反馈：
   - 单击触发 happy
   - 高频点击触发 annoyed
   - 拖拽进入 dragged，释放后恢复 idle

5. 建立环境感知：
   - 根据本地时间影响 sleep 概率
   - 鼠标进入桌宠附近时，头部轻微看向鼠标
   - 所有动作要做平滑过渡

6. 动画控制采用组合式参数叠加：
   breathing / look / react / drag / followMouse
   最终合成为各部件 transform

7. 代码结构清晰，拆分 composables：
   usePetStateMachine
   usePetBehaviorScheduler
   usePetMood
   usePetInteraction
   usePetMotion

8. 注释统一使用中文
9. 保持现有透明窗口与拖拽能力
10. 第一版先不做复杂编辑器，动作参数可直接写在代码中
