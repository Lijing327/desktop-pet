# 《桌面宠物当前版本功能清单 v1（Claude Code 正式执行文档）》

## 一、项目背景

当前项目不是做一个单纯“会动的小宠物”，而是做一个具备 **陪伴感、互动感、生命感** 的桌面宠物 MVP。
当前阶段重点不是商业化全量能力，也不是复杂 AI，而是先完成一个：

**可运行、可演示、可录视频、可继续扩展的桌面宠物基础版本。**

本项目当前已经有方向共识：

* 定位偏 **桌面搭子 / 打工人陪伴**
* 需要体现“活物感”
* 需要支持默认形象选择，后续支持用户自定义形象
* 当前重点是 **交互玩法落地**
* 当前优先实现：

  * 跟随鼠标移动
  * 跑向鼠标时表现为奔跑姿势
  * 等待命令
  * 等待时眼神/朝向跟随鼠标

## 五、核心玩法定义

---

## 5.1 待机模式 Idle

### 目标

让宠物在无操作时也显得“活着”。

### 表现要求

* 原地待机
* 有呼吸感或轻微上下浮动
* 有眨眼动画或轻微小动作
* 偶尔左右看
* 保持自然状态，不僵硬

### 验收标准

* 宠物静止时不会完全像一张死图
* 用户即使不操作，也能看出它处于“活着”的待机状态

---

## 5.2 跟随模式 Follow

### 目标

宠物根据命令进入跟随模式，向鼠标位置移动。

### 表现要求

* 进入跟随状态后，获取鼠标位置
* 当宠物与鼠标有明显距离时，进入跑步/移动状态
* 跑向目标时朝向自动切换
* 到达目标附近后，自动停止并回到待机
* 移动过程尽量平滑，不要瞬移
* 支持基础速度控制
* 支持最小到达距离阈值

### 细化要求

* 距离大：跑步状态
* 距离较小：可直接减速或切换为轻微靠近
* 到达后：切回 Idle
* 方向变化时：角色左右翻转或切换朝向资源

### 验收标准

* 点击“跟随”后，宠物明显开始追向鼠标
* 移动过程是“跑过去”，不是瞬移过去
* 到达后能自然停下

---

## 5.3 等待模式 Wait

### 目标

宠物原地不移动，但会“注视”鼠标，形成陪伴感。

### 表现要求

* 进入等待状态后，不再移动位置
* 身体保持等待动作或站立状态
* 头部、眼睛或整体朝向跟随鼠标位置变化
* 鼠标移动时，宠物有“看过去”的反馈
* 鼠标静止时，逐步回到较自然的等待姿态

### 技术要求

这一版不要求复杂骨骼动画。
可接受的低成本实现方式包括：

* 角色整体左右翻转
* 头部轻微偏移
* 眼睛位置轻微偏移
* 朝向插值变化
* 基于鼠标相对方向的 look-at 视觉模拟

### 验收标准

* 点击“等待”后，宠物不再追着鼠标移动
* 鼠标移动时，宠物看向鼠标方向
* 用户能明显感知“它在看我”

---

## 5.4 睡觉模式 Sleep

### 目标

形成明显状态差异，增强演示效果。

### 表现要求

* 进入睡觉状态后，宠物切换为卧倒/睡觉动作
* 可带有轻微呼吸起伏
* 可以有 Zzz / 泡泡等轻量表现
* 被点击或通过命令可唤醒
* 唤醒后回到 Idle

### 验收标准

* 睡觉状态与待机状态有明显视觉区分
* 用户能一眼看出宠物在休息

---

## 5.5 点击反馈 React

### 目标

增加互动感。

### 表现要求

* 点击宠物后，有短暂反馈动作
* 可选表现：

  * 眨眼
  * 点头
  * 摇头
  * 小跳一下
  * 开心/疑惑小表情
* 点击反馈时长短，不打断主逻辑太久
* 点击反馈结束后回到原状态或 Idle

### 验收标准

* 用户点击宠物时，有明确反馈
* 不会点击后毫无反应

---

## 5.6 拖拽移动 Drag

### 目标

让用户能自定义宠物落点。

### 表现要求

* 鼠标按住宠物可拖动
* 拖动过程中宠物跟随鼠标移动
* 松开后位置固定
* 新位置写入本地配置

### 验收标准

* 宠物能被拖动
* 重启后仍保留最后位置

## 六、状态机设计

请严格按状态机方式组织，不要把行为逻辑写成大量散乱 if else。

### 状态定义

```ts
type PetState =
  | 'idle'
  | 'follow'
  | 'wait'
  | 'sleep'
  | 'drag'
  | 'react'
```

### 状态说明

#### idle

默认状态，待机动画，小幅随机表现

#### follow

接收鼠标位置，朝目标点移动，到达后切 idle

#### wait

原地不动，但朝向/眼神随鼠标变化

#### sleep

睡觉状态，不移动，点击或命令唤醒

#### drag

被用户拖拽时进入，结束后回 idle

#### react

点击反馈短状态，执行完后返回上一个合理状态或 idle

---

### 状态切换规则

#### 外部命令触发

* 托盘菜单“跟随” → follow
* 托盘菜单“等待” → wait
* 托盘菜单“睡觉” → sleep
* 托盘菜单“待机” → idle

#### 用户操作触发

* 鼠标拖拽宠物 → drag
* 点击宠物 → react
* 睡觉状态点击 → idle 或 react 后 idle

#### 自动切换

* follow 到达目标范围 → idle
* react 播放结束 → 恢复到合理状态
* drag 结束 → idle

---

### 状态优先级建议

当多个事件冲突时，建议优先级：

```text
drag > react > sleep > follow > wait > idle
```

或者按实现实际情况微调，但必须统一管理，不允许逻辑冲突。
